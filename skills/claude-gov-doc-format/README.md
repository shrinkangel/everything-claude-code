# claude-gov-doc-format

GB/T 9704-2012 《党政机关公文格式》审核工具 - Claude Code 技能

## 简介

本技能提供公文格式审核和排版指南生成功能，帮助校对公文是否符合国家标准。

## 适用标准

- **GB/T 9704-2012** 《党政机关公文格式》
- 适用于党政机关、企事业单位公文处理

## 使用方法

### `/format-check` - 公文格式审核

审核公文文本的格式规范性。

```typescript
// 输入：待审核的公文文本
/format-check {
  "text": "公文正文内容..."
}
```

**检测项目**：
- 发文字号格式（六角括号 `〔〕`）
- 成文日期格式
- 层级标题结构（一、→（一）→1.→（1））
- 附件标注格式
- 抄送机关格式

**输出示例**：
```
## 公文格式审核报告

### 审核结果：未通过

| 项目 | 数量 |
|------|------|
| 总检查项 | 5 |
| 错误 | 2 |
| 警告 | 1 |

### 发现问题
| 元素 | 问题 | 建议 |
|------|------|------|
| 发文字号 | 使用方括号 [] | 改为六角括号〔〕 |
| 成文日期 | 包含虚位 | 去掉前导零 |
```

### `/layout-gen` - 生成排版指南

根据公文类型生成详细排版参数。

```typescript
// 输入：公文类型
/layout-gen {
  "docType": "general" | "letter" | "command" | "notice" | "report" | "opinion"
}
```

**支持类型**：
- `general` - 通用格式
- `letter` - 信函格式
- `command` - 命令格式
- `notice` - 通知格式
- `report` - 报告格式
- `opinion` - 意见格式

**输出示例**：
```
## 公文排版指南 - 通用格式

### 页面设置
| 参数 | 值 |
|------|-----|
| 幅面 | A4 (210mm × 297mm) |
| 版心 | 156mm × 225mm |
| 天头 | 37mm±1mm |
| 订口 | 28mm±1mm |

### 字体规范
| 元素 | 字体 | 字号 |
|------|------|------|
| 发文机关标志 | 小标宋体 | 2号 |
| 标题 | 小标宋体 | 2号 |
| 正文 | 仿宋体_GB2312 | 3号 |
```

### `/font-check` - 字体字号检查

专门检测正文的字体字号规范。

```typescript
/font-check {
  "text": "公文正文内容..."
}
```

### `/hierarchy-check` - 层级结构检查

检测层级标题顺序和格式规范。

```typescript
/hierarchy-check {
  "text": "公文正文..."
}
```

### `/format-fix` - 自动修正格式

自动修正公文格式问题，并输出修正后的公文。

```typescript
/format-fix {
  "text": "公文内容...",
  "dryRun": false  // true=仅预览不修改
}
```

**可自动修正的问题**：

| 问题类型 | 错误示例 | 修正后 |
|---------|---------|--------|
| 发文字号括号 | `[2024]1号` | `〔2024〕1号` |
| 日期虚位 | `2024年01月01日` | `2024年1月1日` |
| 日期分隔符 | `2024.1.1` | `2024年1月1日` |
| 层级序号格式 | `1)` | `1.` |

**输出示例**：
```
## 格式修正报告

### 修改统计
| 项目 | 数量 |
|------|------|
| 总修改数 | 2 |
| 发文字号修正 | 1 |
| 日期格式修正 | 1 |

### 修改记录
| 行号 | 类型 | 原始内容 | 修正内容 |
|------|------|---------|---------|
| 1 | 发文字号 | [2024]1号 | 〔2024〕1号 |
| 5 | 日期格式 | 2024年01月15日 | 2024年1月15日 |

### 修正后的公文
（修正后的完整公文文本）
```

**预览模式**（dryRun: true）：
- 仅显示修改建议
- 不实际修改原文
- 便于确认修改内容是否符合预期

## 常见错误示例

| 错误类型 | 错误示例 | 正确格式 |
|----------|----------|----------|
| 发文字号括号 | `[2024]1号` | `〔2024〕1号` |
| 成文日期虚位 | `2024年01月01日` | `2024年1月1日` |
| 日期分隔符 | `2024.1.1` | `2024年1月1日` |
| 层级跳跃 | 直接使用 `1.` | 先用 `一、` |
| 一级标题字体 | 使用仿宋体 | 应使用黑体 |
| 二级标题字体 | 使用黑体 | 应使用楷体 |
